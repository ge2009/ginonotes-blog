这篇是用于验证博客发布链路的测试文章，目标是保证「内容可读、页面不报错、提交后可自动部署」。

## 本次测试目标

1. 新增一篇符合仓库规范的 `MDX` 文章
2. 自动配置封面图并在 frontmatter 中引用
3. 本地构建通过（确保线上不会因格式问题报错）
4. 提交并推送到远程仓库

## 与旧版测试内容相比，本次改进

- 增加了 `cover` 字段，避免部分主题在无封面时渲染异常
- 标题、摘要、正文结构都按现有文章风格统一
- 保留固定 slug，便于回归测试和复现

## 当前发布流程（实操版）

```bash
# 1) 新增文章和图片资源
# posts/*.mdx
# public/covers/generated/*.svg

# 2) 本地检查
pnpm build

# 3) 提交与推送
git add .
git commit -m "test: verify blog publishing workflow"
git push origin main
```

## 结果判定标准

如果线上可以正常打开这篇文章，且封面图显示正常，说明以下环节均已通过：

- 内容层（MDX）✅
- 资源层（封面图）✅
- 构建层（Next.js/Contentlayer）✅
- 发布层（Git Push + 自动部署）✅
